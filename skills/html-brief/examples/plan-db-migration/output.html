Plan Brief

Zero-downtime Postgres migration plan

Migrate customer traffic with dual writes, backfill verification, read shadowing, and a reversible cutover. The safest path is expand → backfill → dual-read verification → cutover → contract, with every phase gated by checks.

Overview

Goals

  • Move reads and writes without planned downtime.
  • Keep rollback possible until validation closes.
  • Detect drift before customer impact.

Non-goals

  • No product behavior change.
  • No full data-access rewrite.
  • No destructive cleanup before rollback window closes.

Key decisions

DecisionChoiceRationale
Migration styleExpand and contractSeparates additive changes from cleanup.
Cutover controlFeature flag for read pathAllows rollback without redeploying.
ValidationCounts, checksums, shadow readsCatches correctness drift before traffic moves.

Architecture / Flow

Phased Plan

  1. 1. Expand schema

    Gate: New nullable columns and indexes deployed with no lock-risk operations.

    • Add new nullable structures.
    • Create concurrent indexes.
    • Deploy code that can read old path only.

  2. 2. Dual write

    Gate: Every new write populates both old and new schema.

    • Deploy dual-write code.
    • Emit mismatch metrics.
    • Add retry logging.

  3. 3. Backfill and verify

    Gate: Historical rows are copied and checksums match.

    • Run throttled backfill.
    • Compare counts and checksums.
    • Replay suspicious rows.

  4. 4. Shadow reads and cutover

    Gate: Shadow reads match and read flag can be enabled gradually.

    • Enable shadow reads.
    • Canary read flag.
    • Ramp to all traffic.

  5. 5. Contract old schema

    Gate: Rollback window closed and no consumers read old fields.

    • Remove old reads.
    • Drop dual writes.
    • Remove obsolete columns later.

Risks and mitigations

High Blocking lock during expand

Signal: lock wait alerts. Mitigation: concurrent index creation and small reviewed DDL.

Medium Backfill write amplification

Signal: replication lag. Mitigation: throttle and pause automatically on lag.

High Rollback writes only old schema

Signal: rollback after cutover. Mitigation: keep dual-write compatibility until contract phase.
