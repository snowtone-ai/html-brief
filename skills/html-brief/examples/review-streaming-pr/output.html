Review Brief

Streaming/backpressure PR review

The PR moves response generation to streaming and adds a queue-based backpressure layer, but the cancellation path and high-water behavior need fixes before merge.

1 high1 medium1 lowVerdict: request changes

Motivation

Before

Responses were buffered fully before being sent, increasing latency and memory use.

After

Chunks stream as they arrive, with queue backpressure intended to protect memory.

Change map

Findings

High Abort signal does not stop the producer

Impact: Client disconnects can keep upstream work running and consume memory.

+ for await (const chunk of producer()) { await queue.push(chunk) }

Suggestion: Wire the request AbortSignal through the producer loop and close the queue on abort.

Medium High-water policy can reorder terminal chunks

Impact: A final error or done marker may be dropped behind data chunks, leaving clients hanging.

+ if (queue.length > highWater) queue.shift()

Suggestion: Make terminal events non-droppable and drain or close explicitly.

Low Tests do not cover slow consumers

Impact: Regression risk remains around delayed readers.

Suggestion: Add a fake slow writable and assert bounded queue length.
